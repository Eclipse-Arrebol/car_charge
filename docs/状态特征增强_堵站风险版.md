# 状态特征增强：堵站风险版

## 本次改动目的

固定评估 seed 后可以更清楚地看到，当前 FedDQN 的主要问题不是完全学不会，而是：

- 容易学到“更快到站”
- 但对堵站风险的识别仍不稳定
- 在等待时间与放弃充电指标上仍明显弱于 Greedy

因此本次不再继续硬调 reward，而是优先增强状态特征，让模型更容易直接感知拥堵风险。

## 状态维度变化

节点特征维度由 `15` 扩展为 `18`。

新增特征：

- `Feature[15]`：`queue_wait_ratio`
  - 含义：归一化排队等待比率
  - 计算：`queue_time_h / max_wait_time_h`
  - 作用：让模型直接知道该站等待是否接近超时阈值

- `Feature[16]`：`station_pressure`
  - 含义：站点拥堵压力
  - 计算：`(queue + connected + predicted_arrivals + pending) / (max_queue_len + num_chargers)`
  - 作用：把“队列长度 + 在桩车辆 + 预测到达车辆 + 本步待分配车辆”整合成一个更容易学习的风险量

- `Feature[17]`：`current_target_flag`
  - 含义：当前目标站标记
  - 作用：在受限重规划场景下，告诉模型“当前已经在去哪个站”，方便比较是否值得改派

## 为什么这三个特征更重要

虽然旧状态中已经包含：

- `queue`
- `connected_evs`
- `predicted_arrivals`
- `service_time_h`

但这些信息比较分散，模型需要自己拼出“这个站是不是快堵爆了”。

新增后，模型能更直接地看到：

- 这个站是否接近等待超时
- 这个站当前总体压力是否偏高
- 当前决策是否是一次改派决策

## 代码影响

### 环境侧

位置：

- `env/Traffic.py`

改动：

- `get_graph_state()` 改为输出 18 维节点特征
- `get_graph_state_for_ev()` 为每个站补充新的拥堵风险特征

### 网络侧

位置：

- `agents/GraphQNetwork.py`

改动：

- 队列/容量特征组由原来的 5 维扩展为 8 维
- FeatureEncoder 输入同步更新为 18 维布局

### 训练/评估入口

以下入口的 `num_features` 已统一改为 `18`：

- `main.py`
- `train.py`
- `evaluation/run_evaluation.py`
- `visualization/run_training_real_map.py`
- `visualization/run_training_with_viz.py`
- `visualization/visualize_simulation_3d.py`
- `env/RealTrafficEnv.py`

## 重要注意事项

由于输入特征维度已从 `15` 改为 `18`：

- 旧 checkpoint 不再兼容
- 必须重新训练后再评估

如果继续直接加载旧模型，可能会：

- 维度不匹配报错
- 或得到无意义结果

## 当前预期

本次改动的目标不是立刻让 reward 看起来更高，而是让模型更稳定地区分：

- 近但会堵的站
- 稍远但更稳的站

后续评估时应重点看：

- `avg_wait_time_min`
- `abandoned_evs`
- `evs_in_line`
- `end_evs_in_line`

如果这些指标持续改善，就说明状态增强方向是有效的。
