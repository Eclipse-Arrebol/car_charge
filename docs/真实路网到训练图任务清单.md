# 真实路网到训练图任务清单

## 目标

构建一套可复现、可解释、可映射回真实路网的图抽象流程，用于 FedGRL 的训练与评估。

最终需要形成两层图：

- `L0 真实路网层`：真实可通行道路图，用于真实路径、时间、成本和电网耦合评估
- `L1 训练决策层`：从 L0 规则化简得到的训练图，用于 GNN 决策学习

同时维护两层之间的映射：

- `pi_node: L0节点 -> L1节点`
- `pi_edge: L1边 -> L0真实路径`

---

## 总体原则

1. 不再靠“看起来顺眼”决定简化程度
2. 简化规则必须建立在拓扑语义和物理意义上
3. 训练图必须能映射回真实图
4. 评估指标最终必须回到真实图语义

---

## 阶段一：固定 L0 真实路网层

### 任务 1：固定下载规则

目标：

- 从 OSM 下载真实可通行道路
- 仅保留电动汽车可行驶的主干道路
- 保存标准化 `graphml`

当前建议规则：

- 仅保留 `motorway/trunk/primary/secondary/tertiary` 及部分 `*_link`
- 排除 `access=private`
- 排除 `motor_vehicle=no`
- 排除明显停车场/内部服务道路

当前脚本：

- [download_map.py](/g:/交电demo/map_tools/download_map.py)

输出目录：

- `map_outputs/`

验收标准：

- 生成真实图 `graphml`
- 生成真实图预览 `png`
- 节点和边数量稳定
- 路网不出现明显断裂或大面积缺失

---

## 阶段二：构建 L1 训练决策层

### 任务 2：基于拓扑语义做第一层简化

目标：

- 删除无决策意义的链式中间节点
- 保留关键交叉口、端点和语义锚点

建议规则：

- 删除大部分度为 `2` 的节点
- 保留度不等于 `2` 的节点
- 强制保留：
  - 充电站节点
  - 路网边界节点
  - 主干道路出入口
  - 其他任务相关锚点

要点：

- 这一步的核心是“拓扑语义简化”
- 不要先做大半径空间聚类

验收标准：

- 训练图节点数明显下降
- 但主干拓扑结构仍完整
- 不出现明显假连接

### 任务 3：同一物理交叉口聚合

目标：

- 解决一个真实大路口被拆成多个近邻节点的问题

建议规则：

- 在任务 2 保留的关键节点上再做局部聚合
- 聚合半径使用有物理意义的距离阈值，如 `20m/25m/30m`
- 不用拍脑袋的聚类簇数

要点：

- 这是“物理交叉口尺度”的超参
- 需要后续通过保真指标来选定

验收标准：

- 交叉口附近节点密集现象明显缓解
- 路网骨架仍然与真实图对应

### 任务 4：从真实图重建训练边

目标：

- 不直接使用聚合算法产生的边几何
- 用真实路径重建训练边

建议做法：

- 先确定两个训练节点对应的真实节点簇
- 再回到 L0 中寻找这两个簇之间的真实连接路径
- 用这条真实路径重建 L1 边的几何和长度属性

- 训练图边在可视化上连续自然
- 不出现明显断边、错接或几何异常

## 阶段三：建立映射关系

### 任务 5：显式保存节点映射

目标：

- 保存每个训练节点对应哪些真实节点

至少需要：

- `train_node -> raw_nodes`
- `raw_node -> train_node`

### 任务 6：显式保存边映射

目标：

- 保存每条训练边对应的真实路径

至少需要：

- `train_edge -> raw_path_nodes`
- `train_edge -> raw_edge_sequence`

### 任务 7：显式保存充电站映射

目标：

- 明确充电站在 L0 和 L1 中分别落在哪个节点

至少需要：

- `station_id -> raw_node`
- `station_id -> train_node`

验收标准：

- 给定任一 L1 节点或边，都能追溯回 L0
- 给定任一站点，都能在两层图中唯一定位

---

## 阶段四：做图保真度评估

### 任务 8：最短路径保真度

目标：

- 判断 L1 对 L0 的距离/时间近似是否可靠

建议做法：

- 在 L0 中随机采样若干节点对
- 将它们映射到 L1
- 对比原图最短路和训练图最短路的相对误差

建议统计：

- 平均误差
- 中位数误差
- `95%` 分位数误差

### 任务 9：可达性保持率

目标：

- 判断图抽象后是否破坏连通关系

建议做法：

- 比较 L0 与 L1 上采样节点对的连通性是否一致

### 任务 10：结构重要性保真度

目标：

- 判断 L1 是否保留了关键交通节点的结构角色

建议做法：

- 计算 L1 节点的 betweenness
- 与其对应 L0 节点集合的聚合 betweenness 做相关性分析

建议指标：

- Pearson
- Spearman

验收标准：

- 可以画出“简化强度 vs 保真指标”的曲线
- 找到一组可解释的简化参数，而不是仅凭视觉选参数

---

## 阶段五：把 L1 接入训练

### 任务 11：训练阶段使用 L1 图结构

目标：

- 模型在训练图上学习高层决策

建议：

- `RealTrafficEnv` 接收 L1 训练图作为主图结构
- 节点和边特征先基于 L1 构建

### 任务 12：保留真实代价查询接口

目标：

- 不让训练图脱离真实图语义

建议：

- EV 到站时间、长度、真实路径等核心代价，可以继续由 L0 提供或校准

验收标准：

- 训练图参与决策
- 真实图仍然参与代价计算

---

## 阶段六：评估回到真实图语义

### 任务 13：L1 决策、L0 评估

目标：

- 让论文里的“真实性”和“工程可落地性”成立

具体要求：

- 模型在 L1 上输出动作
- 动作映射回 L0 上的真实站点和真实路径
- 最终指标在 L0 上计算

重点指标：

- 真实行驶时间
- 真实里程
- 真实排队时间
- 真实充电成本
- 真实电网约束

验收标准：

- 可以明确说明“训练在抽象层，评估在真实层”

---

## 推荐实现顺序

按下面顺序推进，避免边训练边改图导致实验漂移：

1. 固定 L0 下载规则
2. 完成 L1 的拓扑简化规则
3. 完成物理交叉口聚合
4. 完成训练边的真实路径重建
5. 完成节点/边/站点映射保存
6. 完成图保真度评估脚本
7. 选择一组正式简化参数
8. 再把 L1 接入训练
9. 评估时回到 L0 计算真实指标

## 最终论文要回答的问题

后续方法和实验的最终目标，不是简单证明“我们用了真实地图”，而是证明：

1. 底层交通数据来自真实可通行路网
2. 训练图是由真实图规则化生成的任务相关语义图
3. 训练图中的节点和边都可映射回真实图
4. 策略在训练图上学习，但在真实图语义下评估

如果这四点都能成立，训练图才真正具备工程上的实践价值。
