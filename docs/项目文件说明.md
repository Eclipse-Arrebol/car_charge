# 项目文件说明

本项目是一个基于图神经网络（GNN）与深度强化学习（DQN）的**电动汽车（EV）充电调度系统**。
系统在一个 3×3 的交通路网仿真中，由智能体学习将待充电 EV 合理分配到两个充电站，
同时兼顾用户平均成本（行驶时间、等待时间、充电费用）与电网安全（电压偏移、线路损耗）。

## 根目录文件

### `main.py`
PyCharm 默认生成的项目入口占位文件，当前仅包含一个 `print_hi()` 示例函数，**不承担实际业务逻辑**。
如需统一入口，可在此文件中集成训练、评估和可视化的调用流程。

### `train.py`
**DQN 训练的核心模块**，包含两部分：

| 类 / 函数 | 说明 |
|---|---|
| `DQNAgent` | DQN 智能体，包含策略网络、目标网络、经验回放缓冲区（Replay Buffer）和 ε-greedy 探索策略 |
| `DQNAgent.select_action()` | 根据当前图状态选择动作（前往哪个充电站） |
| `DQNAgent.store_transition()` | 将 (状态, 动作, 奖励, 下一状态) 元组存入记忆库 |
| `DQNAgent.replay()` | 从记忆库随机采样小批量，用目标网络计算 TD 目标，反向传播更新策略网络 |
| `DQNAgent.save_model()` | 保存模型权重至 `model/trained_dqn.pth` |
| `DQNAgent.load_model()` | 从文件加载已保存的模型权重 |
| `__main__` 训练循环 | 执行 200 个 episode，每个 episode 按 SOC 从低到高对 EV 顺序决策，调用经验回放 |

**关键超参数：**
- 学习率：`0.001`（Adam 优化器）
- 折扣因子 γ：`0.99`
- 目标网络同步间隔：每 `200` 步
- ε 衰减：从 `1.0` 按 `0.995` 衰减到最低 `0.05`
- Replay Buffer 容量：`5000`

## `model/` 模型目录

### `model/GraphQNetwork.py`
**图神经网络 Q 值估计模型**，继承自 `torch.nn.Module`。

**网络结构：**
1. 两层 GCN 卷积（`GCNConv`）：捕获每个节点的空间邻域信息
   - 输入维度：9（节点特征数）→ 32 → 64
2. 全局上下文提取：`global_mean_pool` 对全图做均值池化，得到全局表示向量
3. 充电站节点 Readout MLP：
   - 分别提取两个充电站节点（节点 ID 0 和 8）的嵌入向量
   - 将站节点嵌入（64维）与全局上下文（64维）拼接后，通过两层 MLP 输出每个站对应的 Q 值
4. 最终输出：`[batch_size, 2]` 的 Q 值张量，分别对应两个充电站的动作价值

这种设计使每个充电站的 Q 值基于该站自身的局部特征（排队长度、电价、电压、负荷率等），
避免了全局池化掩盖站间差异的问题。

### `model/trained_dqn.pth`
训练完成后由 `DQNAgent.save_model()` 保存的 PyTorch 模型权重文件，
包含 `policy_net` 状态字典和当前 `epsilon` 值，供评估和推断时加载使用。

## `simEvn/` 仿真环境目录

### `simEvn/Traffic.py`
**项目的核心仿真环境**，实现了交通网络与配电网的联合仿真，遵循 OpenAI Gym 风格接口。

**主要类与函数：**

| 类 / 函数 | 说明 |
|---|---|
| `EV` | 电动汽车实体，维护节点位置、SOC 状态、行驶路径及各类统计指标（行驶步数、等待步数、充电费用等） |
| `EV.move()` | 每步沿路径移动一个节点，同时消耗 2% SOC |
| `get_tou_multiplier()` | 分时电价系数函数，模拟峰（×1.5）/ 平（×1.0）/ 谷（×0.5）三段电价 |
| `PowerGrid` | 简化辐射状配电网模型，计算节点电压（标幺值）和线路损耗 |
| `ChargingStation` | 充电站实体，维护队列、当前充电功率、电价和 CVXPY 凸优化调度接口 |
| `ChargingStation.schedule_power()` | 用 CVXPY 求解充电功率分配的凸优化问题（最大化充电速率，满足功率上限约束） |
| `TrafficPowerEnv` | 主仿真环境类，持有 3×3 交通图、EV 列表、充电站列表和配电网模型 |
| `TrafficPowerEnv.reset()` | 重置环境，随机初始化 EV 位置与 SOC |
| `TrafficPowerEnv.step()` | 执行一步仿真：处理 EV 移动、充电、排队，计算电网状态，返回 `(next_state, reward, done, info)` |
| `TrafficPowerEnv.get_graph_state()` | 将当前仿真状态编码为 PyG `Data` 对象（图状态），作为网络输入 |
| `TrafficPowerEnv.get_graph_state_for_ev()` | 为指定 EV 构建带 EV 指示位的个性化图状态（含 pending_counts 感知） |

**节点特征（9维）：**

| 维度 | 含义 |
|---|---|
| 0 | 当前节点的 EV 数量 |
| 1 | 是否为充电站（0/1） |
| 2 | 充电站队列长度（非充电站为 0） |
| 3 | 充电站当前电价（非充电站为 0） |
| 4 | 充电站当前负荷率（非充电站为 0） |
| 5 | 配电网节点电压（标幺值） |
| 6 | 时间步编码（归一化） |
| 7 | 分时电价系数 |
| 8 | EV 指示位（当前决策 EV 所在节点为 1，其余为 0） |

**奖励函数：** 综合考虑充电功率收益（正向）、电压偏移惩罚、队列长度惩罚和线路损耗惩罚。

## `evaluation/` 评估目录

### `evaluation/metrics.py`
**评价指标计算模块**，提供用户侧和电网侧的量化评估指标。

| 类 | 说明 |
|---|---|
| `CostParams` | 集中管理成本参数：时间价值（VOTT = 30 CNY/h）、步时长、购电基准价格 |
| `UserMetrics` | 从 EV 对象列表提取用户侧指标，包括行驶/等待/充电时间成本、充电费用、总成本、平均等待时间等 |
| `GridMetrics` | 从电网历史数据提取电网侧指标，包括配电网购电成本和累计电压偏移量 |
| `Evaluator` | 聚合类，在仿真过程中持续记录电网状态，episode 结束后一次性输出完整评估报告 |

### `evaluation/run_evaluation.py`
**评估入口脚本**，训练完成后运行，支持两种模式：

- **随机策略基线**（`use_random=True`）：以随机动作为基准，评估未经训练时的性能
- **DQN 策略评估**（`use_random=False`）：加载 `model/trained_dqn.pth`，以 ε=0 纯利用模式跑评估 episode

运行方式：
```bash
python evaluation/run_evaluation.py
```
结果输出至 `evaluation/results/evaluation_report.json`。

### `evaluation/results/evaluation_report.json`
评估结果的 JSON 格式输出文件，包含多轮评估的用户侧与电网侧指标均值。

## `visualization/` 可视化目录

### `visualization/run_training_with_viz.py`
**带可视化功能的训练入口**，与 `train.py` 中的训练逻辑相同，
额外集成了 `TrainingVisualizer` 进行训练过程数据记录与图表生成，
是推荐的训练运行方式。

运行方式：
```bash
python visualization/run_training_with_viz.py
```

### `visualization/visualize_training.py`
**训练过程可视化工具类 `TrainingVisualizer`**，负责：
- 逐 episode 记录奖励、ε 值、平均队列长度、电网过载次数等数据
- 训练结束后绘制奖励曲线（含滑动窗口均值）、ε 衰减曲线
- 将训练数据序列化保存至 `visualization/results/training_data.json`
- 生成训练摘要文本 `visualization/results/training_summary.txt`

### `visualization/visualize_network.py`
**交通网络状态可视化类 `NetworkVisualizer`**，以 NetworkX + Matplotlib 渲染当前仿真步的图状态：
- 节点颜色表示 EV 数量，充电站节点使用特殊形状标注
- 显示各节点的队列长度、电价信息
- 可将图像保存为文件或直接展示

### `visualization/visualize_performance.py`
**训练后性能分析类 `PerformanceAnalyzer`**，读取 `training_data.json`，
计算并绘制：
- 奖励的最大值、最小值、均值、标准差和趋势斜率
- 奖励与 ε 的对比折线图
- 各辅助指标（队列长度、过载次数等）的历史曲线

### `visualization/visualize_simulation.py`
**仿真动画演示模块**，以启发式策略（队列 + 电价加权最小的站）驱动 EV 决策，
使用 Matplotlib Animation 生成逐步动态可视化动画，
可输出 GIF/MP4 文件，用于直观展示仿真场景。

### `visualization/results/`
可视化和训练过程输出目录：
- `training_data.json`：JSON 格式的每 episode 训练数据（奖励、ε、队列等）
- `training_summary.txt`：训练结束后自动生成的文字摘要报告

## 运行流程

```
1. 训练模型
   python visualization/run_training_with_viz.py
   （或直接 python train.py，无可视化输出）

2. 评估模型（随机基线）
   python evaluation/run_evaluation.py
   （修改 use_random=False 可评估 DQN 策略）

3. 查看结果
   - 训练曲线图：visualization/results/
   - 评估报告：  evaluation/results/evaluation_report.json
```
