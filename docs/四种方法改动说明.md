# 四种方法改动说明

## 方法概览

| 方法 | 主要改动 | 目标 |
|------|----------|------|
| Voltage Buggy | 原始电压奖励函数（有bug） | 基准对比 |
| Voltage Fixed | 修复abandon penalty bug | 解决reward hacking |
| Voltage New | 调整奖励权重 user:grid = 0.7:0.3 | 解决稀疏事件dilution |
| Mask New | 添加action mask机制 | 强制约束，避免爆队 |

## 1. Voltage Buggy（原始电压奖励函数）

### 代码位置
- `training/config.py`: `voltage_user_weight = 0.3`, `voltage_grid_weight = 0.7`
- `training/trainer.py`: 默认权重 0.3:0.7

### 奖励函数设计
```python
# 电压奖励模式
user_cost = travel_time + wait_time + charging_fee
grid_cost = voltage_excursion + line_losses

# 权重配置（Buggy版本）
weighted_sum = 0.3 * user_norm + 0.7 * grid_norm
```

### 问题
1. **Reward Hacking**：policy 通过让 EV 排队超时来降低电压偏差
2. **abandoned_evs = 1.2**：仍有 EV 放弃充电
3. **queue_time_h = 0.099**：排队时间较高

### 实验结果
| 指标 | 数值 |
|------|------|
| avg_wait_time_min | 247.4 |
| abandoned_evs | 1.2 |
| voltage_excursion | 22.5 |

## 2. Voltage Fixed（修复abandon penalty bug）

### 改动内容
**修复bug**：`env/charging_station.py` 中的 `transition` 补存逻辑

### 代码改动
```python
# 修复前：abandon transition 未正确存储
if ev.just_abandoned_this_step:
    # bug: transition 未保存到 replay buffer

# 修复后：正确存储 abandon transition
if ev.just_abandoned_this_step:
    self.store_transition(state, action, reward, next_state, action_mask)
```

### 奖励函数（同Buggy）
```python

```

### 改进效果
| 指标 | Buggy | Fixed | 改进 |
|------|-------|-------|------|
| abandoned_evs | 1.2 | 0.8 | -33% |
| queue_time_h | 0.099 | 0.074 | -25% |

### 问题仍未解决
- **abandoned_evs = 0.8**：仍高于理想值
- **稀疏事件 dilution**：abandon 事件稀疏，reward 信号被淹没

## 3. Voltage New（调整奖励权重）

### 改动内容
**调整奖励权重**：`user:grid = 0.7:0.3`（user 主导）

### 代码改动
```python
# training/config.py
voltage_user_weight: float = 0.7  # 原为 0.3
voltage_grid_weight: float = 0.3  # 原为 0.7

# training/trainer.py
DEFAULT_VOLTAGE_USER_WEIGHT = 0.7  # 原为 0.3
DEFAULT_VOLTAGE_GRID_WEIGHT = 0.3  # 原为 0.7
```

### 奖励函数调整
```python
# 调整后：user 成本主导
weighted_sum = 0.7 * user_norm + 0.3 * grid_norm
```

### 设计动机
1. **解决稀疏事件 dilution**：abandon 事件稀疏，需要更强的 user 成本惩罚
2. **避免 reward hacking**：减少 grid 信号的主导作用
3. **一步到位**：直接跳到 0.7:0.3，节省 GPU 时间

### 改进效果
| 指标 | Fixed | New | 改进 |
|------|-------|-----|------|
| avg_wait_time_min | 232.3 | 186.2 | -20% |
| abandoned_evs | 0.8 | 0.1 | -87% |
| total_charging_cost | 733.2 | 692.9 | -5.5% |

### 关键发现
- **abandoned_evs 从 0.8 降至 0.1**：user 权重调整显著改善
- **wait_time 从 232.3 降至 186.2**：综合性能提升
- **成本降低 5.5%**：user 主导后成本优化更好

## 4. Mask New（添加action mask机制）

### 改动内容
**添加action mask**：基于队列长度屏蔽爆队站点

### 代码改动
```python
# env/base_env.py
def get_action_mask(self, ev, pending_counts=None):
    mask = torch.ones(1, num_actions, dtype=torch.bool)
    
    for i, station in enumerate(self.stations):
        # 队列过长时屏蔽
        if len(station.queue) >= threshold:
            mask[0, i] = False
    
    return mask
```

### mask 触发条件
```python
# 队列长度阈值（可配置）
queue_threshold = station.max_queue_len * 0.8
```

### 设计动机
1. **解决稀疏事件 dilution**：直接屏蔽爆队站点，不依赖 reward 信号传播
2. **强制纠正**：早期训练时强制 policy 避开爆队站点
3. **加速收敛**：减少无效探索

### 改进效果
| 指标 | Voltage New | Mask New | 差异 |
|------|-------------|----------|------|
| avg_wait_time_min | 186.2 | 192.1 | +3.2% |
| abandoned_evs | 0.1 | 0.8 | +700% |
| queue_time_h | 0.0055 | 0.0047 | -15% |

### 关键发现
- **queue_time 更低**：mask 有效降低排队时间
- **abandoned_evs 更高**：mask 限制了 policy 探索空间
- **权衡关系**：hard constraint vs soft reward shaping

## 5. Mask Time（总时间mask）

### 改动内容
**改进mask触发条件**：基于总时间（行程+排队）屏蔽

### 代码改动
```python
# env/base_env.py
total_time_h = metrics["trip_time_h"] + metrics["queue_time_h"]
threshold_h = 1.0  # 总时间阈值（小时）
should_mask = total_time_h > threshold_h
```

### 设计动机
1. **更合理的约束**：考虑行程时间，避免"远+排队久"的站点
2. **用户体验保护**：总等待时间超过阈值才屏蔽
3. **早期纠正**：训练早期强制纠正不良选择

### 实验结果（step17）
- threshold = 1.0h（更严格）
- 预期：mask触发率更高，queue_time更低

## 方法对比总结

### 性能排名（综合）
1. **Voltage New**：等待时间最低，弃单EV最少，成本适中
2. **Mask New**：排队时间最低，但弃单EV较高
3. **Voltage Fixed**：修复bug后性能提升
4. **Voltage Buggy**：原始版本，存在reward hacking

### 论文叙事建议
> "我们逐步改进奖励函数和约束机制：首先修复abandon penalty bug（Voltage Fixed），然后调整奖励权重解决稀疏事件dilution（Voltage New），最后引入action mask机制强化约束（Mask New）。实验表明Voltage New在综合性能上最优，而Mask New在降低排队时间方面更有效，体现了hard constraint与soft reward shaping的权衡关系。"
