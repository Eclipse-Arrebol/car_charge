# 真实路网训练图交接文档

## 1. 当前目标

为 FedGRL 构建一套可复现的真实路网图抽象流程：

- `L0 真实路网层`：真实可通行主干路网
- `L1 训练决策层`：由 L0 规则化简得到的训练图
- 维护 `L0 -> L1` 的节点映射，保证训练图仍可追溯到真实图

当前工作重点不是接入训练，而是先把“真实图 -> 训练图 -> 评估判据”这条链路做扎实。

当前 `L1` 的节点语义采用“代表锚点”口径：

- 每个 cluster 在 `L0` 中选择一个代表节点 `r(cluster)`
- `L1` 节点位置使用该代表节点坐标
- 映射可理解为 `pi(u) = r(cluster(u))`

这样 `L1` 的节点不再是纯虚拟质心，而是 `L0` 节点子集上的语义代表点。

## 2. 当前结论

当前冻结基线目录：

- [baseline_eps40_artifacts](</g:/交电demo/map_outputs/baseline_eps40_artifacts>)

后续如果要接训练、做论文配图或复现实验，优先从这个目录取文件，而不是再从 `map_outputs/` 根目录零散查找。

### 2.1 推荐默认配置

基于固定采样对和统一 fidelity envelope 的参数扫描，当前推荐：

- `eps* = 40m`

采用的约束为：

- `short cross-cluster P95 <= 25%`
- `medium cross-cluster P95 <= 10%`
- `long cross-cluster P95 <= 10%`
- `short same-cluster rate <= 10%`

在满足上述约束的前提下，`eps=40m` 具有最大压缩率，因此被选为当前默认推荐值。

### 2.2 当前最重要判断

1. `L0` 图在下载阶段已经被 OSMnx 做过简化，因此 `Topology` 层基本是 `no-op`
2. 真正起主要压缩作用的是 `L1` 的局部聚合
3. 之前 fidelity 异常高的主要原因是评估脚本把 `pi(u) == pi(v)` 的 pair 直接算成了 100% 误差
4. 修复该口径后，当前图抽象结果已经进入“参数折中”阶段，而不是“实现 bug”阶段

## 6. 已定位并修复的问题

### 6.1 fidelity 评估口径 bug

旧问题：

- 若 `pi(u) == pi(v)`，会出现 `d1=0`
- 导致 `err = 1.0`
- 使 short 桶 P95 人工飙高

修复后：

- `same-cluster` pair 单独统计
- 主结果使用 `cross-cluster metrics`
- 同时输出 `same_cluster_rate` 和 `intra_cluster_stats`

### 6.2 segment 多候选边问题

已经做过专项诊断：

- 当前这版图中，每个 `cluster-pair` 只有 `1` 条 segment
- 因此当前版本不存在“多候选边被错误取 min”的实际触发问题

备注：

- 这是当前数据上的结论
- 若后续更换城市或更换下载规则，建议重新跑一次 segment 诊断

## 7. 建议的下一步

### 7.1 如果继续图抽象主线

优先顺序：

1. 将 `eps=40` 固化为默认训练图配置
2. 整理正式的层间映射结构
3. 把 `L0/L1` 的说明写进方法章节

### 7.2 如果准备接入训练

建议顺序：

1. 先固定 `L1 = eps40` 这张图
2. 在环境里明确：
   - 训练使用 `L1`
   - 真实代价和最终评估回到 `L0`
3. 先跑通一版最小训练实验，再决定是否需要继续优化图抽象

### 7.3 当前不建议继续做的事

- 不建议继续无上限地扫描更多 `eps`
- 不建议继续反复人工看图调参数
- 不建议在未固定 `L1` 前就接训练和评估，否则实验口径会漂
