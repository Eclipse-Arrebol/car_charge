# 项目架构说明

## 1. 项目简介

本项目面向电动汽车充电调度问题，目标是在动态交通与配电网约束下，为低电量 EV 选择合适的充电站。  
训练部分采用联邦强化学习，将图神经网络 DQN 部署到多个客户端环境中，通过 FedAvg/FedProx 聚合参数。  
环境层支持真实路网仿真，`RealTrafficEnv` 基于 OSMnx 路网构建交通图，并复用充电站、电网、奖励与评估逻辑。

## 3. 模块说明

### 3.1 `agents/`

#### `agents/network.py`

- `FeatureEncoder`
  - 作用：将 `Traffic.py` 输出的 18 维节点特征按语义分组编码。
  - 分组：
    - 队列/容量组
    - 价格/时间组
    - 空间/电网组
    - EV 状态组
  - 核心方法：
    - `forward(x)`：输入 `[N, 18]`，输出编码后的节点特征。

- `GraphQNetwork`
  - 作用：图神经网络 Q 函数，结构为 `FeatureEncoder -> GATv2 x 2 -> Dueling Head`。
  - 关键点：
    - 使用 `station_node_ids` 从整张图中抽取各充电站节点表示。
    - 同时构造全局图表示 `global_mean_pool`。
    - 用 `Q = V(s) + A(s,a) - mean(A)` 输出每个站点的 Q 值。
  - 核心方法：
    - `forward(data, action_mask=None)`：支持动作掩码，将非法动作置为极小值。

#### `agents/dqn_base.py`

- `DQNBase`
  - 作用：为单机 DQN 与联邦客户端提供统一训练基础设施。
  - 内部组件：
    - `policy_net` / `target_net`
    - Adam 优化器
    - replay buffer
    - epsilon-greedy 行为策略
  - 核心方法：
    - `select_action(state_data, action_mask=None)`：按 epsilon-greedy 选动作。
    - `store_transition(state, action, reward, next_state, action_mask=None)`：存经验。
    - `decay_epsilon()`：按衰减率更新探索率。
    - `_prepare_batch(minibatch)`：将经验打包为 PyG `Batch`。
    - `_compute_td_loss(...)`：计算 Double DQN TD loss。
    - `_apply_gradients(loss, batch_size=None)`：默认梯度更新逻辑。
    - `_sync_target()`：周期同步目标网络。
    - `train_on_batch(batch_size)`：完整执行一次采样与训练。

#### `agents/FederatedDQN.py`

- `FederatedClient(DQNBase)`
  - 作用：联邦学习客户端，对应一个本地环境。
  - 在 `DQNBase` 基础上增加：
    - FedProx 近端正则
    - DP-SGD 梯度裁剪与高斯噪声
    - 本轮样本计数与隐私预算统计
  - 核心方法：
    - `store_transition(...)`：额外统计本轮样本数。
    - `_apply_gradients(loss, batch_size=None)`：注入 FedProx / DP-SGD。
    - `local_train(batch_size, num_steps=1)`：执行若干步本地训练。
    - `load_global_model(global_state_dict)`：加载全局模型并记录锚点参数。
    - `get_model_params()`：返回本地参数供聚合。
    - `get_privacy_spent()`：查询累计隐私预算。
    - `reset_round_counter()`：联邦轮开始前清空样本计数。

- `FederatedServer`
  - 作用：维护全局模型并执行参数聚合。
  - 核心方法：
    - `register_client(client)`：注册客户端。
    - `distribute_global_model()`：向所有客户端下发全局参数。
    - `aggregate()`：按 `num_samples_this_round` 执行 FedAvg。
    - `save_global_model(path)`：保存全局模型。
    - `load_global_model(path)`：加载全局模型并同步到客户端。

### 3.2 `env/`

#### 继承关系

```text
TrafficPowerEnv
└── RealTrafficEnv
```

#### 模块拆分关系

```text
entities.py          -> EV
power_grid.py        -> get_tou_multiplier / PowerGrid / _FlexiblePowerGrid
charging_station.py  -> ChargingStation
base_env.py          -> TrafficPowerEnv
real_env.py          -> RealTrafficEnv
Traffic.py           -> 兼容垫片
RealTrafficEnv.py    -> 兼容垫片
```

#### `env/entities.py`

- `EV`
  - 作用：描述单辆电动车的状态、移动过程与评估统计字段。
  - 核心方法：
    - `move(env, step_hours=1.0)`：沿路网边逐步推进车辆位置，并更新行驶时间与 SOC。

#### `env/power_grid.py`

- `get_tou_multiplier(time_step, steps_per_day=24)`
  - 作用：返回当前时间步的分时电价系数。

- `PowerGrid`
  - 作用：基础辐射状配电网模型，计算母线电压与线路损耗。
  - 核心方法：
    - `run_power_flow(loads)`

- `_FlexiblePowerGrid(PowerGrid)`
  - 作用：支持任意数量充电站的电网拓扑扩展版本。
  - 用途：由 `RealTrafficEnv` 在多站配置下使用。

#### `env/charging_station.py`

- `ChargingStation`
  - 作用：封装单个充电站的队列管理、动态电价、到达预测与功率分配。
  - 核心方法：
    - `update_price(tou_multiplier=1.0, price_noise=0.0)`
    - `estimate_queue_wait_hours(incoming_count=0)`
    - `estimate_average_charge_hours()`
    - `estimate_charge_time_hours(ev)`
    - `estimate_charge_cost(ev)`
    - `update_arrival_prediction(arrivals_this_step)`
    - `optimize_power()`
    - `step(tou_multiplier=1.0, price_noise=0.0)`

#### `env/base_env.py`

- `TrafficPowerEnv`
  - 作用：
    - 定义基础交通-充电-电网联合仿真环境。
    - 维护 EV、充电站、配电网、图状态构造与奖励逻辑。
  - 主要接口：
    - `reset()`
    - `get_pending_decision_evs()`
    - `estimate_action_metrics(ev, station_id, pending_counts=None)`
    - `get_graph_state()`
    - `get_graph_state_for_ev(ev, pending_counts=None)`
    - `get_action_mask(ev)`
    - `step(actions)`
    - `render()`
  - 其他重要辅助方法：
    - `should_request_charge_decision(ev)`
    - `enter_edge(u, v)` / `leave_edge(u, v)`
    - `_build_edge_index()`
    - 一组动态交通时间与容量估计函数，例如 `_dynamic_profiles()`、`_travel_time_weight()`。

#### `env/real_env.py`

- `RealTrafficEnv(TrafficPowerEnv)`
  - 作用：将基础 3x3 网格替换为真实城市路网。
  - 特点：
    - 支持从 `graphml_file` 直接加载本地图。
    - 支持按地名或经纬度下载/裁剪路网。
    - 暴露 `num_nodes`、`station_node_ids`，供 GNN 初始化使用。
  - 核心方法：
    - `__init__(...)`：加载路网、初始化站点、电网与 EV。
    - `reset()`：重建 EV 与站点状态，但保留真实路网结构。

#### `env/Traffic.py` 与 `env/RealTrafficEnv.py`

- 当前作用：兼容垫片。
- 用途：为旧 import 路径提供 re-export，避免重构后立刻打断现有调用方。

### 3.4 `evaluation/`

#### Strategy 类图（文字版）

```text
BaseStrategy
├── RandomStrategy
├── GreedyStrategy
├── DQNStrategy
└── FedDQNStrategy
```

- `BaseStrategy`
  - `select_action(env, ev, pending_counts) -> int`
  - `name() -> str`

- `RandomStrategy`
  - 基于动作掩码在可行动作中随机选择。

- `GreedyStrategy`
  - 按归一化负载最轻站点选动作。
  - 负载由队列长度、已连接车辆与当前待分配数共同组成。

- `DQNStrategy`
  - 内部持有 `DQNAgent`。
  - 初始化时根据环境的 `num_stations`、`station_node_ids`、`num_nodes` 构建网络并加载权重。

- `FedDQNStrategy`
  - 继承 `DQNStrategy`，仅通过 `name()` 区分为联邦模型。

#### `evaluation/run_evaluation.py`

- 核心函数签名：

```python
def evaluate(
    strategy: BaseStrategy,
    eval_cfg,
    episode_seeds: list,
) -> dict:
```

- 兼容入口签名：

```python
def run_evaluation(
    episodes=50,
    steps_per_episode=1000,
    use_random=False,
    use_greedy=False,
    use_real_map=True,
    model_file=None,
    num_evs=100,
    num_stations=4,
    episode_seeds=None,

```

- 辅助函数：
  - `_build_eval_env(eval_cfg, seed)`
  - `_resolve_strategy(use_random, use_greedy, model_file, env)`
  - `_compare_table(reports)`

#### `evaluation/metrics.py`

- `CostParams`：统一管理时间价值、电价与步长参数。
- `UserMetrics`：统计用户侧成本与等待指标。
- `GridMetrics`：统计电网购电成本、电压偏移、线损。
- `Evaluator`
  - `reset()`
  - `update(info, power_grid, stations=None)`
  - `report(evs, stations=None, verbose=True)`

## 5. 数据流说明

训练调用链从 `main.py` 开始：命令行参数首先被解析为 `TrainConfig`，随后 `cmd_train_real()` 调用 `training.trainer.run_training_real()`，再由 `FederatedTrainer` 初始化两个 `RealTrafficEnv`、一个 `FederatedServer` 和多个 `FederatedClient`。每个 step 中，环境通过 `get_pending_decision_evs()` 与 `get_graph_state_for_ev()` 生成图状态，客户端调用 `DQNBase.select_action()` 基于 `GraphQNetwork` 选站，`env.step(actions)` 执行动作并返回奖励与电网信息，训练器将经验写入回放池并进行本地训练。每个 episode 结束后，`FederatedServer.aggregate()` 汇总客户端参数，`distribute_global_model()` 下发新全局模型，训练器记录可视化数据、按周期保存 checkpoint，最终将全局模型保存到 `checkpoints/` 并输出训练摘要。
