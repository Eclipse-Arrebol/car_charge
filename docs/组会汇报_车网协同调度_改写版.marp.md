# 研究背景

- 电动汽车保有量持续增长，充电需求呈现出更强的时空集中性与动态波动性。
- 在实际城市运行中，车辆的充电决策并不只是“去哪充电”，还会同时影响：
  - 道路侧：绕行距离、拥堵传播、到站时间；
  - 站点侧：排队长度、服务效率、资源利用率；
  - 电网侧：局部负荷聚集、电压越限风险、线路损耗水平。
- 因此，电动汽车充电问题本质上不是单一的路径规划问题，也不是单一的电价优化问题，而是一个交通系统与配电系统深度耦合的动态调度问题。

**核心认识：同一个充电决策，会同时作用于“车—路—站—网”多个子系统。**

---

# 现有研究不足

- 一类工作主要从交通侧出发，关注最近站点选择、最短路径或排队时间预测，但对配电网运行约束考虑不足。
- 一类工作主要从电网侧出发，关注削峰填谷、价格引导或负荷转移，但对车辆真实出行行为与路网状态刻画较弱。
- 还有不少研究基于规则网格路网或小规模理想场景，难以反映真实城市路网中的复杂拓扑和路径代价差异。

**现有方法的共同问题：**

> 往往只优化某一个子系统，缺乏面向真实路网的车-路-站-网协同决策能力。

---

# 为什么做这个研究方向

## 现实层面：单侧优化已经难以适应高渗透率场景

- 当电动汽车数量较少时，“最近站点优先”或“低电价优先”还能基本满足需求；
- 但在高渗透率条件下，大量车辆集中前往局部区域，容易引发排队累积、道路拥堵和负荷冲击；
- 因而，充电调度必须从“单站推荐”升级为“交通-充电-电网协同调度”。

## 研究层面：这是一个强耦合动态决策问题

- 系统状态来源多样，包含车辆状态、道路拓扑、站点排队和电网负荷等异构信息；
- 单次动作会带来跨系统连锁影响，具有明显的时序性、随机性和耦合性；
- 这类问题难以用单一静态优化模型完整描述，更适合图建模与强化学习结合处理。

## 工程层面：需要从理想模型走向真实城市场景

- 如果方法只能在简化规则路网上有效，则工程参考价值有限；
- 只有将算法放到真实 OSM 路网环境中，才能检验其可扩展性与实用性；
- 因此，本研究不仅关注策略性能，也强调场景真实性和执行可行性。

---

# 研究目标与关键科学问题

## 研究目标

构建一个面向真实路网环境的电动汽车充电协同调度框架，实现以下目标：

- 为电动汽车动态选择合适充电站；
- 在站端容量约束下进行可执行的功率调度；
- 在保证用户服务体验的同时兼顾配电网运行安全；
- 为不同区域路网提供可迁移的训练与评估流程。

## 关键科学问题

1. 如何统一表示“车-路-站-网”的动态耦合状态？  
2. 如何在用户效率、站点均衡与电网安全之间建立合理权衡？  
3. 如何保证强化学习策略不仅能学出来，而且能真正执行？  
4. 如何将方法从小规模仿真推广到真实城市路网场景？

---

# 总体技术路线

本研究采用“上层选站 + 下层调度 + 电网反馈”的分层协同框架：

1. **交通侧建模**：基于真实道路网络刻画车辆出行过程与路径代价；  
2. **决策侧建模**：利用图强化学习方法输出充电站选择策略；  
3. **站端执行层**：通过凸优化进行站内充电功率分配；  
4. **电网反馈层**：将站点负荷映射到配电网节点，并计算负荷、电压与损耗指标。  

最终形成“状态感知—策略决策—站端执行—电网反馈”的闭环优化流程。

---

# 方法框架设计

## 上层：图强化学习选站

- 利用图神经网络编码真实路网与充电站分布的拓扑关系；
- 将车辆位置、SOC、候选站点状态等信息映射到统一图结构中；
- 采用 Double DQN、目标网络与经验回放机制提高训练稳定性；
- 引入动作掩码，过滤不可达或不满足约束的无效动作。

## 下层：站内凸优化调度

- 在站点容量约束、单桩功率约束下分配充电功率；
- 协调不同车辆的充电效率与服务公平性；
- 提升策略从“选站决策”到“实际执行”的可落地性。

---

# 状态、动作与奖励设计

## 状态设计：体现车-路-站-网耦合

- **车辆侧**：当前位置、SOC、剩余出行需求；
- **道路侧**：路网拓扑、路段属性、路径通行代价；
- **站点侧**：排队长度、服务车辆数、实时价格；
- **电网侧**：节点负荷、电压偏移、线路损耗等运行指标。

## 动作设计

- 为待决策电动汽车选择目标充电站；
- 动作空间通过约束过滤减少无效探索。

## 奖励设计思路

- 出行时间与等待时间惩罚：提升用户效率；
- 充电费用惩罚：控制综合成本；
- 队列不均衡惩罚：抑制局部拥堵；
- 电压越限惩罚：约束电网安全风险；
- 谷时利用激励：引导柔性负荷向友好时段转移。

**奖励函数的核心目标，是将“用户体验”与“系统安全”共同编码进策略学习过程。**

# 创新点

## 创新点 1：面向真实路网的充电协同建模

- 相比基于规则网格或抽象拓扑的研究，本研究引入真实 OSM 路网构建交通环境；
- 能够更真实地反映道路连通性、路径代价与空间分布差异；
- 使充电调度研究从理想场景走向真实城市网络。

## 创新点 2：构建车-路-站-网统一耦合状态表示

- 将车辆状态、道路拓扑、站点排队和电网运行指标纳入统一状态空间；
- 避免传统方法只依赖局部信息进行单侧优化；
- 使策略能够显式感知跨系统耦合影响。

## 创新点 3：图强化学习与物理约束反馈融合

- 利用图神经网络学习真实路网中的结构依赖关系；
- 将电压偏移、负荷风险等电网物理约束通过奖励反馈融入策略学习；
- 实现“空间拓扑建模 + 物理约束引导”的结合。

## 创新点 4：分层协同机制提升方法可执行性

- 针对很多学习型方法停留在“推荐哪个站”层面的不足；
- 本研究采用“上层选站 + 下层凸优化调度”的分层框架；
- 使高层策略能够进一步落实到具体的站内功率分配方案上。

## 创新点 5：为后续分布式扩展预留联邦训练接口

- 面向多区域、多运营主体场景下的数据隔离与协同需求；
- 在框架层面预留联邦聚合扩展路径；
- 为后续研究多主体协同训练提供结构基础。

# 实验设置

- **环境构建**：真实城市路网 + 充电站分布 + 简化配电网模型；
- **训练配置**：训练 500 轮，探索率从 1.0 衰减至 0.05；
- **评估指标**：奖励收敛、排队与等待、电压安全、综合成本等；
- **训练摘要**：
  - 平均奖励：-273.88
  - 最近 10 轮平均奖励：-257.60
  - 平均队列长度：0.03
  - 总电压越限次数：0

---

# 结果分析与阶段性认识

- 从训练曲线看，模型已从随机探索逐步过渡到稳定策略学习；
- 从排队结果看，模型具备一定的站点导流与负载分散能力；
- 从电网指标看，当前奖励设计对高风险行为具有一定抑制作用；
- 当前实验下未出现明显电压越限，说明安全约束反馈具备初步有效性；
- 但等待时间和服务成本仍有进一步优化空间，说明站端资源匹配与导流策略还需继续改进。

**阶段性结论：当前工作已初步验证“车-路-站-网”闭环框架的可行性，并明确了后续性能提升方向。**

# 研究价值

## 理论价值

- 为交通—电力耦合场景下的动态决策问题提供统一建模思路；
- 探索图强化学习在真实路网充电调度中的适用性；
- 为“数据驱动策略学习 + 物理约束反馈”结合提供研究基础。

## 方法价值

- 构建了可扩展的车-路-站-网闭环优化框架；
- 兼顾空间拓扑建模、策略学习与执行层调度。

## 应用价值

- 可为充电运营方提供兼顾用户体验与电网安全的调度支持；
- 对后续开展城市级充电协同管理研究具有参考意义。

---

# 当前不足与下一步工作

## 当前不足

- 现阶段基线对比仍不充分，尚不能系统证明相对优势；
- 配电网模型仍为简化形式，对复杂运行场景覆盖有限；
- 价格机制与需求随机性建模仍可进一步增强；
- 联邦扩展目前仍处于框架预留阶段，尚缺少完整实验支撑。

## 下一步工作

1. 增加传统策略与强化学习基线对比；  
2. 补充消融实验，分析动作掩码、站内优化与电网惩罚项的贡献；  
3. 扩展场景规模，验证更多车辆、更多站点与更复杂路网下的表现；  
4. 完善电网建模与反馈机制，提高电压与负荷约束刻画精度；  
5. 推进联邦训练实验，分析性能与通信开销之间的权衡关系。

---

# 结论与展望

- 本研究聚焦真实路网条件下的电动汽车充电协同调度问题；
- 构建了融合图强化学习、站端优化与电网反馈的分层闭环框架；
- 初步实验表明，该框架在队列控制和电网安全约束方面具有一定可行性；
- 下一步将通过更完整的对比实验、消融实验和复杂场景扩展，进一步验证方法优势并提升工程应用能力。
