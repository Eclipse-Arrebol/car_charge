# 状态构造与约束分析 - 组会汇报材料

## 一、状态构造（18维节点特征）

### 1.1 特征维度定义

| 索引 | 特征名称 | 含义 | 更新时机 |
|------|----------|------|----------|
| 0 | ev_count | 节点上的EV数量 | 每步更新 |
| 1 | is_station | 是否为充电站节点 | 固定 |
| 2 | queue | 充电站队列长度 | 每步更新 |
| 3 | price | 充电站当前价格 | 每步更新 |
| 4 | connected | 连接充电桩的EV数量 | 每步更新 |
| 5 | load | 充电站负载比例 | 每步更新 |
| 6 | voltage | 配电网节点电压 | 每步更新 |
| 7 | tou | 分时电价乘数 | 每步更新 |
| 8 | soc | **当前EV的SOC** | 每决策更新 |
| 9 | inv_dist | 1/(1+行程时间) | 每决策更新 |
| 10 | trip_time | 行程时间(h) | 每决策更新 |
| 11 | service_time | 服务时间(h) | 每决策更新 |
| 12 | gen_cost | 广义成本/100 | 每决策更新 |
| 13 | price_noise | 价格噪声 | 每步更新 |
| 14 | pred_arrivals | 预测到达量(EMA) | 每步更新 |
| 15 | queue_wait_ratio | 队列等待比例 | 每步更新 |
| 16 | station_pressure | 站点压力指数 | 每步更新 |
| 17 | charger_avail | 可用充电桩比例 | 每步更新 |

### 1.2 特征分组（FeatureEncoder）

**四组语义特征**：
1. **队列/容量组** [1,2,4,5,14,15,16,17]：站点拥堵状态
2. **价格/时间组** [3,7,10,11,12,13]：成本与时间信号
3. **空间/电网组** [0,6,9]：流量、电压、距离
4. **EV状态组** [8]：请求车辆SOC

### 1.3 关键设计

#### EMA预测到达量（feat[14]）
```python
station.predicted_arrivals  # 指数移动平均预测
```
- 作用：让模型提前感知拥堵风险
- 更新：每步基于实际到达量更新

#### 站点压力指数（feat[16]）
```python
station_pressure = (queue + connected + predicted_arrivals) / max_capacity
```
- 作用：综合评估站点负载压力
- 范围：0~1+，超过1表示超负荷

#### 队列等待比例（feat[15]）
```python
queue_wait_ratio = queue_wait_hours / max_wait_time_h
```
- 作用：标准化等待时间，便于模型学习
- 裁剪：min(2.0, ratio) 避免极端值

## 二、约束条件（Action Mask）

### 2.1 约束类型

| 约束类型 | 条件 | 掩码操作 |
|----------|------|----------|
| **路径可达性** | `nx.has_path(traffic_graph, ev.curr_node, station.traffic_node_id)` | 不可达 → mask=False |
| **SOC约束** | `ev.soc < soc_needed` | 电量不足 → mask=False |
| **行程时间约束** | `trip_time_h >= 24.0` | 行程过长 → mask=False |
| **超时约束** | `total_time_h > threshold_h` | 总时间超阈值 → mask=False |

### 2.2 超时约束详解

```python
total_time_h = trip_time_h + queue_time_h
threshold_h = 1.0  # 可配置，默认1小时
should_mask = total_time_h > threshold_h
```

**设计动机**：
- 避免EV选择"远+排队久"的站点
- 保护用户体验（总等待时间）
- 早期强制纠正，后期policy内化

### 2.3 回退机制

当所有动作都被mask时：
```python
if not mask.any():
    mask.fill_(True)  # 恢复所有动作
```

**统计指标**：
- `all_masked_before_fallback`：所有动作被mask的次数
- `partial_masked_ratio`：部分动作被mask的比例

## 三、组会汇报要点

### 3.1 状态构造创新点

1. **多维度异质特征**：18维特征涵盖交通、电力、用户状态
2. **语义分组编码**：四组特征独立编码后融合，避免异质特征混杂
3. **预测能力**：EMA预测到达量赋予模型拥堵预判能力
4. **标准化设计**：队列等待比例、站点压力指数等标准化特征

### 3.2 约束设计优势

1. **物理合理性**：SOC约束、路径可达性符合现实逻辑
2. **用户体验保护**：超时约束避免极端等待时间
3. **早期纠正**：mask机制在训练早期强制纠正不良选择
4. **自适应学习**：后期policy内化约束，mask触发率下降

### 3.3 实验验证

从step15实验结果看：
- `partial_masked_ratio` 从ep1的15%降至ep10的8%
- 说明policy正在主动学会避开爆队站点
- mask机制有效且policy已内化约束

### 3.4 论文叙事建议

**状态构造**：
> "我们设计了18维异质节点特征，涵盖交通状态、电力状态、用户状态三方面。通过语义分组编码（FeatureEncoder）避免异质特征直接混入GAT，提升模型表达能力。"

**约束设计**：
> "我们引入action mask机制，基于路径可达性、SOC、行程时间、总等待时间四重约束，保护用户体验并加速训练收敛。实验表明mask触发率随训练下降，说明policy已内化约束。"
