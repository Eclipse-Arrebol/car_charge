# 固定评估 Seed 说明

## 为什么要固定评估 Seed

在当前项目的评估过程中，环境初始化包含随机因素，例如：

- EV 初始位置
- EV 初始 SOC
- 每轮 episode 的场景初始化

如果每次评估都使用不同的随机种子，那么即使模型完全没有变化，评估结果也可能出现明显波动。

这会导致一个问题：

- 无法判断“结果变化”到底来自模型改进，还是只是来自随机场景变化

因此，当前项目已改为默认使用固定评估 seed。

## 当前实现方式

评估入口：

- `main.py -> cmd_evaluate`

当前固定的基准 seed：

```python
DEFAULT_EVAL_BASE_SEED = 20260411
```

评估时会：

1. 先使用 `DEFAULT_EVAL_BASE_SEED`
2. 基于它生成一组固定的 `episode_seeds`
3. Random / Greedy / DQN / FedDQN 四种策略共用同一组 episode seeds

这样做可以保证：

- 每次评估面对的是同一批场景
- 不同策略之间比较公平
- 同一个 checkpoint 多次评估结果可复现

## 如何理解

可以把固定 seed 理解为：

- 不固定 seed：每次考试题都换
- 固定 seed：每次都做同一套题

固定 seed 不是取消随机性，而是让随机性可复现。

## 当前训练/评估实践建议

### 用途 1：稳定对比不同模型

当你比较：

- 不同 checkpoint
- 不同 reward 设计
- 不同重规划机制

时，建议始终使用固定评估 seed。

这样看到的差异更可信。

### 用途 2：论文主结果

在论文主结果中，建议：

- 使用固定 seed 做主表对比
- 再补充多 seed 平均结果作为稳健性验证

## 注意事项

固定 seed 适合做：

- 可复现实验
- 不同策略公平对比
- 改动前后效果判断

但如果你想评估模型对不同随机场景的泛化能力，仍应额外做：

- 多组不同 seed 的平均实验

## 当前日志输出

评估时会打印：

```text
[Evaluation Seeds] base_seed=20260411, episode_seeds=[...]
```

看到这行，说明当前评估已进入固定 seed 模式。
