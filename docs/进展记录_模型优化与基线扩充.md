## 1. 本轮 medium 评估结果

运行 `python main.py evaluate --medium`（20 轮均值）后的四方对比：

| 指标 | Random | Greedy | DQN | FedDQN |
|------|--------|--------|-----|--------|
| avg_wait_time_min | 215.25 | **158.70 (-26%)** | 181.35 (-16%) | 176.25 (-18%) |
| abandoned_evs | 38.00 | **25.65 (-32%)** | 32.10 (-16%) | 29.55 (-22%) |
| end_evs_in_line | 0.25 | 0.15 (-40%) | 0.15 (-40%) | **0.10 (-60%)** |
| total_charging_cost_per_veh | 1099.50 | **1099.29 (-0%)** | 1120.28 (+2%) | 1114.24 (+1%) |

**关键结论：**
- FedDQN 的 wait_time 现在优于 DQN（-18% vs -16%），联邦训练有效
- Greedy 在 wait_time 和 abandoned_evs 上击败了两个 RL 模型，说明 reward 对等待时间的激励不足
- charge_time 对所有模型均有所上升（+3~4%），EV 被导向拥堵少但可能更远的站

## 2. 新增 Greedy 基线

**文件改动：**
- `evaluation/run_evaluation.py`
- `main.py`

**实现逻辑（`_greedy_action`）：**

对每辆待决策的 EV，遍历所有可用充电站，选取综合负载最小的站：

```
负载分数 = (queue 数 + connected 数 + 本步 pending 数) / num_chargers
```

**接入方式：**
- `run_evaluation()` 新增 `use_greedy=True` 参数
- `main.py cmd_evaluate` 改为四方对比（Random / Greedy / DQN / FedDQN）

**运行命令不变：**
```bash
python main.py evaluate --medium
```

## 3. GraphQNetwork 架构改进

**文件：** `agents/GraphQNetwork.py`

> ⚠️ 架构已改变，旧 checkpoint 不兼容，需重新训练。

### 3.1 新增 FeatureEncoder（分组特征编码）

原版将 15 维原始节点特征直接输入 GAT，存在两个问题：
1. 各特征量纲差异极大（队列数 0~20、电压 pu ~1.0、广义成本 0~100+），GAT 无法有效区分权重
2. 异质特征语义混杂，难以学习

新方案将 15 维特征按语义分 4 组，各组独立编码后融合：

| 组 | 特征索引 | 维度 | 语义 |
|----|---------|------|------|
| 队列/容量 | 1,2,4,5,14 | 5 | is_station, queue, connected, load, predicted_arrivals |
| 价格/时间 | 3,7,10,11,12,13 | 6 | price, TOU, travel_time, service_time, gen_cost, price_noise |
| 空间/电网 | 0,6,9 | 3 | ev_count, voltage, inv_distance |
| EV 状态 | 8 | 1 | 请求车辆 SOC |

各组分别经 `Linear(→32) + ReLU`，再拼接后经 `Linear(128→64) + LayerNorm + ReLU` 输出 64 维。

### 3.2 Dueling Network 读出头

原版对每个站各跑一路 MLP 直接输出 Q 值。

新版改为标准 Dueling 架构：
- **Value stream** `V(s)`：从全局 context 估计当前状态整体价值
- **Advantage stream** `A(s,a)`：对每个站节点估计相对优势
- 合并：`Q(s,a) = V(s) + A(s,a) − mean_a(A(s,a))`

好处：V stream 负责"现在整体好不好"，A stream 负责"哪个站更好"，梯度解耦，训练更稳定。

### 3.3 完整新架构

```
原始节点特征 [N, 15]
    ↓  FeatureEncoder（4组编码 + LayerNorm）
编码特征 [N, 64]
    ↓  GATv2Conv(64→32, heads=4)
    ↓  GATv2Conv(32→64, heads=4)
节点 embedding [N, 64]
    ↓  global_mean_pool → global_ctx [B, 64]
    ↓  站节点 readout
Advantage [B, num_actions]  +  Value [B, 1]
    ↓  Dueling 合并
Q 值 [B, num_actions]
    ↓  Action Mask
最终 Q 值
```
