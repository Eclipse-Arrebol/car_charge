## 2. GraphQNetwork 架构改进

**文件：** `agents/GraphQNetwork.py`（完全重写）

> ⚠️ 架构已改变，旧 checkpoint 不兼容，需重新训练。

### 2.1 FeatureEncoder（分组特征编码）

将 15 维原始节点特征按语义分 4 组独立编码后融合，解决原版量纲差异大、语义混杂的问题：

| 组 | 索引 | 维度 | 语义 |
|----|------|------|------|
| 队列/容量 | 1,2,4,5,14 | 5 | is_station, queue, connected, load, predicted_arrivals |
| 价格/时间 | 3,7,10,11,12,13 | 6 | price, TOU, travel_time, service_time, gen_cost, price_noise |
| 空间/电网 | 0,6,9 | 3 | ev_count, voltage, inv_distance |
| EV 状态 | 8 | 1 | 请求车辆 SOC |

各组 `Linear(→32) + ReLU`，拼接后 `Linear(128→64) + LayerNorm + ReLU` 输出 64 维。

### 2.2 Dueling Network 读出头

- **Value stream** `V(s)`：全局 context → 当前状态整体价值
- **Advantage stream** `A(s,a)`：每个站节点 → 相对优势
- 合并：`Q(s,a) = V(s) + A(s,a) − mean_a(A(s,a))`

### 2.3 完整架构

```
原始节点特征 [N, 15]
    ↓  FeatureEncoder（4组编码 + LayerNorm）→ [N, 64]
    ↓  GATv2Conv(64→32, heads=4)
    ↓  GATv2Conv(32→64, heads=4)
    ↓  global_mean_pool → global_ctx [B, 64]
    ↓  Dueling Head
      ├─ Value:     global_ctx → V(s) [B, 1]
      └─ Advantage: station_emb + global_ctx → A(s,a) [B, 1] × num_stations
Q = V + A − mean(A)  →  Action Mask  →  输出
```

## 5. Opacus 差分隐私（DP-SGD）

**文件：** `agents/FederatedDQN.py`、`visualization/run_training_real_map.py`、`main.py`

### 5.1 实现方案

采用手动 DP-SGD（兼容 PyG GATv2Conv）+ Opacus `RDPAccountant` 追踪隐私预算：

1. `loss.backward()` 后裁剪梯度至范数 C
2. 对每个参数注入高斯噪声 `N(0, (σ·C/batch_size)²)`
3. `RDPAccountant.step()` 累积隐私消耗，训练结束打印 ε

### 5.2 新增参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `use_dp` | `False` | 是否启用 DP |
| `dp_noise_multiplier` σ | `1.0` | 噪声倍率，越大隐私越强 |
| `dp_clip_C` | `1.0` | 梯度裁剪范数 |
| `dp_delta` | `1e-5` | (ε,δ)-DP 中的 δ |
| `dp_sample_rate` | `batch/20000` | 泊松采样率 q |

### 5.3 运行命令

```bash
# 普通联邦训练（FedProx，无 DP）
python main.py train-real --medium

# 带差分隐私
python main.py train-real --medium --dp
```

训练结束后自动打印：
```
[差分隐私] 训练结束后的隐私预算:
  Client 0: ε=8.xxxx, δ=1e-05
  Client 1: ε=8.xxxx, δ=1e-05
```
