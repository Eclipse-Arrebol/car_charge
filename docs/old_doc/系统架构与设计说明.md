# 系统架构与设计说明

本项目是一个基于**图神经网络 (GNN)** 与**深度强化学习 (DRL)** 的电动汽车 (EV) 充电导航与调度系统。系统在一个真实的城市路网（交通网）和配电网（电网）耦合仿真环境中，通过智能体（Agent）学习如何将待充电的 EV 引导至最合适的充电站，以平衡用户的时间和经济成本，并同时保证配电网的安全稳定运行。

系统架构呈现出宏观（RL 路径及站点推荐）与微观（凸优化站内功率分配）相结合的特征。

---

## 1. 模型架构设计 (Model Architecture)

在状态表示与动作估值层面，项目采用了定制化的 **GNN Q-Network** (`model/GraphQNetwork.py`)，使其不再局限于固定拓扑的 3×3 网格，能够泛化到任意节点数的真实路网。

- **状态输入 (State Representation)**: 
  提取路网的节点特征（包含位置、交通排队状况、电价、电网负荷率、电压偏移等）以及边的邻接矩阵 (`edge_index`)，构建为 PyTorch Geometric 的 `Data` 图对象。
- **特征提取特征卷层 (GCN Layers)**: 
  使用两层图卷积 (`GCNConv`) 捕捉空间邻域信息，将节点特征维度从 9 维映射到 32 维，再到 64 维的高维隐空间。
- **全局上下文与站节点融合 (Readout Mechanism)**:
  - 使用 `global_mean_pool` 获取整张图的全局上下文向量 (64维)。
  - 从图中**精确提取出各个候选充电站节点的嵌入向量** (64维)。
  - 将各个站节点的嵌入与全局向量进行拼接组合 (128维)，通过 MLP (多层感知机) 为每一个充电站进行 Q 值打分。
- **输出 (Output)**: 
  输出形状为 `[Batch Size, 充电站数量]` 的离散 Q 值张量，表示选择前往每个充电站的长期期望收益。

## 2. 价值函数与奖励设计 (Value Function & Reward)

强化学习模块基于经典的 **DQN (Deep Q-Network)** 算法。为了实现多目标的系统级优化，奖励函数（价值函数的设计核心）在 `simEvn/Traffic.py` 中被设计成一个结合了交通、经济与电网安全的复合函数：

1. **队列均衡惩罚 (Queue Imbalance)**：惩罚充电站之间的排队时长差距（如 `-3.0 * |负荷差|`），以此引导车辆分散，避免“羊群效应”。
2. **总等待时长惩罚 (Total Waiting)**：对所有充电站的积压排队车辆数进行惩罚，优化用户整体的时间成本。
3. **充电费用惩罚 (Dynamic Charging Cost)**：基于站点当前的动态电价（受分时电价和拥堵附加费影响），惩罚去高价站充电的行为，引导用户节约成本。
4. **谷时利用奖励 (Valley-time Utilization)**：在电网的分时低谷期，鼓励提高变压器的容量利用率，辅助削峰填谷。
5. **电网电压越限惩罚 (Voltage Violation)**：如果分配导致局部电网节点电压跌落至安全阈值以下，给予严重惩罚（如 `-5.0` /次）。

通过上述设计，DQN 的价值函数隐式地逼近了**“用户时间最短+电费最低+电网最安全”**的帕累托前沿。

## 3. 环境的搭建建模 (Environment Modeling)

项目构建了高度物理保真的车-网耦合环境 (CPTN)，分为交通网和电网两部分：

### 3.1 真实城市路网环境 (`simEvn/RealTrafficEnv.py` & `osm_loader.py`)
系统摒弃了传统的静态网格，通过 **OSMnx** 接入真实世界 GIS 数据（如“珠江新城”或某市某区的离线 `.graphml` 路网文件）：
- **动态图生成**：根据给定的经纬度和范围，自动下载或加载缓存路网，将其转换为 NetworkX 有向图，再映射为 PyTorch Geometric 可用的张量。
- **路径导航**：EV 根据 Dijkstra 或 A* 算法，在真实的节点之间依据物理连接一步步进行寻路与移动。

### 3.2 配电网潮流与拓扑 (`simEvn/Traffic.py`)
为了逼真模拟充电对电网的冲击，环境内置了一个简化的配电网（Power Grid）模型：
- **潮流计算 (Power Flow)**：根据各站节点的实时负载需求，运用电压降落公式（$v_{pu} = 1.0 - \frac{P \cdot R}{U^2}$）及线路损耗计算。
- **安全约束**：实时监控各节点的电压（$v_{pu}$ 是否超出正常范围）和变压器的最高配额。

## 4. 充电站的个数与选点策略

### 4.1 充电站宏观选点与个数设置
项目在初始化时（如 `RealTrafficEnv` 中），允许自由传入 `num_stations` 控制充电站的个数。具体选点策略由 `osm_loader.py` 实现：
- **基于真实拓扑重要性选址**：系统在加载真实路网后，会计算路网中各个节点的**度中心性 (Degree / Traffic hubs)**，自动挑出连通度最高、最核心的 $N$ 个路口作为电动汽车充电站（CS）的物理锚点，使充电设施贴近真实的车流汇聚地。

### 4.2 充电站内部微观功率分配 (CVXPY 凸优化)
除了宏观层面的路由分配，每个充电站在接收到车辆后的底层调度采用凸优化求解（`ChargingStation.optimize_power()`）：
- 目标：最大化当前已连接 EV 的总充电功率，并附带正则项保证多车之间的功率分配公平性。
- 约束：总功率严格不能越过站点的变压器容量上限（`max_grid_power`），单车功率不跨越充电桩物理上限（`max_charger_power`）。
- 这种 **"上层 RL 导流寻路 + 下层 CVX 分配功率"** 的双层设计，是本系统解决复杂约束下车网协同调度的核心亮点。
