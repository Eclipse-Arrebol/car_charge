## 快速上手

### 第一步：下载真实路网（只需做一次）

确保本机代理运行在 `127.0.0.1:7890`，然后：

```powershell
cd "G:\交电demo"
python download_map.py
```

成功后在项目根目录生成 `zhujiang_new_town.graphml`，并弹出路网预览图。

### 第二步：训练

编辑 `visualization/run_training_real_map.py` 顶部配置区，将 `LOCAL_GRAPHML` 指向刚下载的文件：

```python
LOCAL_GRAPHML = r"G:\交电demo\zhujiang_new_town.graphml"
```

然后运行：

```powershell
python visualization/run_training_real_map.py
```

训练结果保存至 `visualization/results_real/`，模型保存至 `model/trained_dqn_real.pth`。

## 三种路网来源模式

在 `visualization/run_training_real_map.py` 配置区选择：

| 模式 | 配置 | 是否需要联网 | 是否真实路网 |
|---|---|---|---|
| **A 自动联网** | 默认（`LOCAL_GRAPHML=None`，`OFFLINE_FALLBACK=False`） | ✓ 尝试 3 个 Overpass 服务器，每个超时 20 秒 | ✓ |
| **B 本地文件** | `LOCAL_GRAPHML = "路径/xxx.graphml"` | ✗ | ✓ |

**推荐模式 B**：先用 `download_map.py` 下载一次，之后无需联网即可反复训练。

## 各文件详细说明

### `download_map.py`

基于你测试可用的代理抓取真实路网，执行以下流程：

1. `ox.graph_from_point(center, dist, custom_filter)` —— 只抓主干道（primary / secondary / tertiary），排除小区内道路，大幅减少节点数
2. `ox.project_graph(G)` —— 投影到平面坐标系（单位：米），用于 `consolidate_intersections` 距离计算
3. `ox.consolidate_intersections(tolerance=15, dead_ends=False)` —— 合并 15 米范围内的复杂路口，去除断头路
4. `ox.add_edge_speeds / add_edge_travel_times` —— 补充速度与时间属性
5. `ox.save_graphml(G_proj, filepath)` —— 导出 `.graphml` 文件

**可修改的配置（文件顶部）：**

```python
CENTER_POINT    = (23.1193, 113.3213)   # 纬度, 经度（当前：广州珠江新城）
RADIUS_METERS   = 3000                  # 抓取半径（米）
CONSOLIDATE_TOL = 15                    # 路口合并容差（米）；越大节点越少
OUTPUT_FILE     = "zhujiang_new_town.graphml"
PROXY           = "http://127.0.0.1:7890"
```

> **注意**：`consolidate_intersections(rebuild_graph=True)` 会产生 `frozenset` 类型的节点 ID，`osm_loader.py` 内部已通过 `nx.convert_node_labels_to_integers()` 处理，无需手动干预。

### `simEvn/osm_loader.py`

路网加载的核心工具库，对外暴露三个函数：

#### `load_road_network(place, ...)`

通过 OSM 地名下载路网。依次尝试三个 Overpass 服务器，每个超时 20 秒，全部失败自动降级为离线合成路网。

```python
from simEvn.osm_loader import load_road_network

graph, station_nodes, positions = load_road_network(
    place="Wuchang District, Wuhan, China",
    num_stations=2,
    max_nodes=30,
    offline=False,   # True = 直接用合成路网
)
```

#### `load_road_network_from_file(filepath, ...)`

从本地 `.graphml` 或 `.osm` 文件加载，处理流程：

1. 加载文件（`ox.load_graphml` 或 `ox.graph_from_xml`）
2. 转无向图 + 整数化节点 ID（兼容 frozenset）
3. 取最大连通分量
4. 若节点数 > `max_nodes`，用 **BFS 采样**精简（见下方说明）
5. 选取充电站节点（高度节点，互相保持距离）
6. 结果 pickle 缓存至 `cache/` 目录

> **坐标系说明**：`download_map.py` 导出的文件坐标为投影坐标系（单位米，数值 >10000），加载时**不做反投影**——RL 训练中路径计算基于图拓扑而非实际坐标，坐标系不影响训练结果。

#### `load_road_network_by_point(lat, lon, dist_m, ...)`

经纬度中心点方式下载，联网失败自动降级，用法同 `load_road_network`。

#### BFS 采样算法（`_sample_nodes`）

旧版采用 KMeans 选空间代表点，在路网稀疏区域会产生互不相连的孤立节点，导致最大连通分量退化为 1 个节点。

**改进方案**：BFS 从全图度数最大的节点出发，优先扩展度数大的邻居（走主干道），收集满 `max_nodes` 个节点后停止。这些节点通过 BFS 树天然连通，子图保证连通。

```
起点 = argmax(degree)
BFS 队列，邻居按 degree 降序扩展
收集满 max_nodes 个节点 → 取子图 → convert_node_labels_to_integers
```

### `simEvn/RealTrafficEnv.py`

继承 `TrafficPowerEnv`，将 3×3 网格替换为任意路网，**所有仿真逻辑（EV 移动、CVXPY 调度、分时电价）完全复用**。

**构造参数：**

| 参数 | 类型 | 说明 |
|---|---|---|
| `place` | str | OSM 地名（模式 A） |
| `graphml_file` | str | 本地文件路径（模式 B，优先级最高） |
| `offline` | bool | True = 离线合成路网（模式 C） |
| `lat`, `lon`, `dist_m` | float / int | 经纬度 + 半径（模式 A 的替代） |
| `num_stations` | int | 充电站数量 |
| `num_evs` | int | EV 数量 |
| `max_nodes` | int | 路网精简节点上限 |

**对外暴露的属性：**

```python
env.num_nodes          # 实际路网节点数，供 DQNAgent 初始化使用
env.station_node_ids   # 充电站节点 ID 列表，供 GraphQNetwork 注册 buffer
env.num_stations       # 充电站数量（= num_actions）
```

---

### `model/GraphQNetwork.py`（修改）

新增对任意路网规模的支持：

- `station_node_ids` 由构造参数传入，通过 `register_buffer` 注册（保存/加载模型时自动携带）
- `num_nodes_per_graph` 支持任意节点数（原来硬编码为 9）

```python
agent = DQNAgent(
    num_features=9,
    num_actions=env.num_stations,
    station_node_ids=env.station_node_ids,   # 真实路网的充电站节点 ID
    num_nodes_per_graph=env.num_nodes,        # 真实路网的节点数
)
```

---

### `train.py`（修改）

`load_model` 改为 `strict=False` 加载，兼容旧版 checkpoint（不含 `station_node_ids` buffer）：

```python
missing, unexpected = self.policy_net.load_state_dict(
    checkpoint['policy_net'], strict=False
)
```

旧模型加载时，`station_node_ids` 保留构造函数中的默认值 `[0, 8]`（3×3 网格）。

## 常见问题

### Q：运行时节点数输出为 1

**原因**：旧版 `_sample_nodes` 用 KMeans 选的 30 个节点在图中互不相连，最大连通分量退化为 1 个节点。

**解决**：已改为 BFS 采样，重跑前需删除旧缓存：

```powershell
Remove-Item "G:\交电demo\cache\local_*.pkl" -Force
```

---

### Q：加载旧模型报 `Missing key: station_node_ids`

**原因**：旧版 `GraphQNetwork` 没有 `register_buffer`，保存的 checkpoint 不含该 key，严格模式加载报错。

**解决**：`train.py` 已改为 `strict=False`，直接重新运行评估即可。

---

### Q：联网下载超时

**原因**：Overpass 官方服务器在中国大陆网络受限。

**解决方案（三选一）**：
1. 开启代理，使用 `download_map.py` 下载 `.graphml` 后用模式 B
2. 设 `OFFLINE_FALLBACK = True` 使用合成路网（模式 C）
3. 模式 A 会自动尝试三个镜像服务器（kumi.systems → mail.ru → overpass-api.de），每个超时 20 秒
