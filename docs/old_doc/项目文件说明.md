# 项目文件说明

本项目是一个面向电动汽车充电调度的强化学习实验系统。核心目标是在交通网与配电网耦合环境中，为低电量 EV 选择合适的充电站，兼顾用户侧成本与电网侧安全。

当前项目已经按“源码、环境、权重、结果、数据”重新整理，入口统一为 `main.py`，训练、评估、可视化三条链路分离但互相兼容。

---

## 1. 项目定位

系统包含三层核心能力：

1. 环境层：构建交通路网、车辆状态、充电站排队、电价、电网负载等仿真对象。
2. 决策层：使用图神经网络 + DQN / Federated DQN 输出“去哪个充电站”的离散动作。
3. 评估层：对随机策略、普通 DQN、联邦 DQN 的调度效果做统一比较。

支持两类场景：

- 合成 3×3 路网：用于快速调试与基础训练。
- 真实 GraphML 路网：用于接近真实交通结构的训练、评估和可视化。

## 3. 架构分层说明

### 3.1 入口层

`main.py` 是当前唯一推荐入口，负责分发不同任务。

支持命令：

```bash
python main.py train
python main.py train-real
python main.py train-viz
python main.py evaluate
python main.py download-map
```

这层只做流程分发，不承载核心算法实现。

### 3.2 智能体层

`agents/` 目录负责所有与模型参数、Q 网络、联邦聚合有关的逻辑。

- `GraphQNetwork.py`
   图神经网络 Q 函数近似器。输入图状态，输出每个候选充电站的 Q 值。

- `FederatedDQN.py`
   提供联邦学习版本的训练结构：
   - `FederatedServer` 维护全局模型。
   - `FederatedClient` 在本地经验上训练，再做 FedAvg 聚合。

### 3.3 环境层

`env/` 目录负责仿真对象和交互规则。

- `Traffic.py`
   基础环境，包含 EV、充电站、简化电网、排队与功率调度等逻辑。

- `RealTrafficEnv.py`
   在 `Traffic.py` 基础上接入真实路网，保留同样的调度接口。

- `osm_loader.py`
   负责加载 GraphML、联网下载 OSM 数据、离线缓存，以及站点候选节点生成。

### 3.4 训练层

训练逻辑分为两条主线：

- `train.py`
   提供标准 DQN 智能体 `DQNAgent`，适合基础训练与加载推理。

- `visualization/run_training_real_map.py`
   面向真实路网联邦训练，带训练过程记录和权重保存。

### 3.5 评估层

`evaluation/run_evaluation.py` 是统一评估脚本，当前支持三类策略：

1. Random Baseline
2. DQN
3. Federated DQN

评估输出包括：

- 控制台对比表
- `results/evaluation_report.json` 评估结果文件

### 3.6 可视化层

`visualization/` 目录只负责展示与记录，不直接定义环境规则。

主要作用：

- 绘制训练曲线
- 分析训练数据
- 展示路网状态
- 回放车辆在路网中的移动过程

## 4. 关键文件职责

### `main.py`

统一入口。优先使用它，而不是直接手动运行分散脚本。

### `train.py`

包含标准 DQN 智能体：

- 策略网络 / 目标网络
- 经验回放
- ε-greedy 动作选择
- 模型保存与加载

同时保留一个基础训练循环，适合合成路网实验。

### `agents/GraphQNetwork.py`

项目核心模型。使用图结构作为状态输入，而不是普通向量状态，因此可迁移到更复杂的路网拓扑。

### `agents/FederatedDQN.py`

项目的扩展训练路线。适合模拟多个站点区域分别学习、再由中心服务器做模型聚合的场景。

### `env/Traffic.py`

是整个项目最关键的业务环境文件之一，决定了：

- 状态长什么样
- 动作如何生效
- 奖励如何计算
- EV 如何移动
- 站点如何排队和充电
- 电网约束如何反馈到奖励与指标

### `env/RealTrafficEnv.py`

将环境从规则网格扩展到真实城市路网，是“从玩具环境到真实结构实验”的关键桥梁。

### `evaluation/run_evaluation.py`

承担两件事：

1. 独立评估某个策略权重
2. 输出 Random / DQN / FedDQN 的横向对比表

## 5. 训练与评估数据流

### 5.1 基础 DQN 训练流

```text
环境 env.Traffic / env.RealTrafficEnv
      -> 生成图状态
      -> DQNAgent.select_action()
      -> 环境 step(actions)
      -> 存储 transition
      -> replay() 更新策略网络
      -> 保存到 checkpoints/
```

### 5.2 联邦 DQN 训练流

```text
RealTrafficEnv
      -> 多个 FederatedClient 各自收集经验
      -> 本地更新 policy_net
      -> FederatedServer.aggregate() 做 FedAvg
      -> 保存全局模型到 checkpoints/
```

### 5.3 评估流

```text
选择环境
      -> 选择策略(Random / DQN / FedDQN)
      -> 逐步执行动作
      -> Evaluator 累积指标
      -> 输出控制台对比表
      -> 写入 results/evaluation_report.json
```

## 6. 当前目录整理原则

本次整理后的目录设计遵循以下原则：

1. `agents/` 只放模型定义，不再混放权重文件。
2. `checkpoints/` 只放 `.pth` 权重。
3. `env/` 统一承载环境逻辑，不再使用旧的 `simEvn/` 命名。
4. `results/` 统一承载训练与评估产物，不再散落在多个子目录。
5. `data/cache/` 专门承载缓存数据，和源码分离。
6. `main.py` 作为单一入口，避免脚本入口分散。

这意味着项目结构已经从“实验堆叠式目录”转为“按职责拆分的工程化目录”。

## 8. 后续建议

如果继续整理，下一步最值得做的是：

1. 将 `train.py` 中的训练循环继续拆分到 `training/` 目录，和 `DQNAgent` 类解耦。
2. 给 `main.py` 增加参数化命令行选项，例如 episode 数、是否使用真实路网、权重文件名。
3. 把评估输出细分为 `results/evaluation/`，进一步区分不同实验批次。
4. 为真实路网、基础网格、联邦训练分别建立更明确的配置文件。

当前版本已经适合继续开发与实验，不再是原来那种源码、权重、结果混放的结构。

这种设计使每个充电站的 Q 值基于该站自身的局部特征（排队长度、电价、电压、负荷率等），
避免了全局池化掩盖站间差异的问题。

## `simEvn/` 仿真环境目录

### `simEvn/Traffic.py`
**项目的核心仿真环境**，实现了交通网络与配电网的联合仿真，遵循 OpenAI Gym 风格接口。

**主要类与函数：**

| 类 / 函数 | 说明 |
|---|---|
| `EV` | 电动汽车实体，维护节点位置、SOC 状态、行驶路径及各类统计指标（行驶步数、等待步数、充电费用等） |
| `EV.move()` | 每步沿路径移动一个节点，同时消耗 2% SOC |
| `get_tou_multiplier()` | 分时电价系数函数，模拟峰（×1.5）/ 平（×1.0）/ 谷（×0.5）三段电价 |
| `PowerGrid` | 简化辐射状配电网模型，计算节点电压（标幺值）和线路损耗 |
| `ChargingStation` | 充电站实体，维护队列、当前充电功率、电价和 CVXPY 凸优化调度接口 |
| `ChargingStation.schedule_power()` | 用 CVXPY 求解充电功率分配的凸优化问题（最大化充电速率，满足功率上限约束） |
| `TrafficPowerEnv` | 主仿真环境类，持有 3×3 交通图、EV 列表、充电站列表和配电网模型 |
| `TrafficPowerEnv.reset()` | 重置环境，随机初始化 EV 位置与 SOC |
| `TrafficPowerEnv.step()` | 执行一步仿真：处理 EV 移动、充电、排队，计算电网状态，返回 `(next_state, reward, done, info)` |
| `TrafficPowerEnv.get_graph_state()` | 将当前仿真状态编码为 PyG `Data` 对象（图状态），作为网络输入 |
| `TrafficPowerEnv.get_graph_state_for_ev()` | 为指定 EV 构建带 EV 指示位的个性化图状态（含 pending_counts 感知） |

**节点特征（9维）：**

| 维度 | 含义 |
|---|---|
| 0 | 当前节点的 EV 数量 |
| 1 | 是否为充电站（0/1） |
| 2 | 充电站队列长度（非充电站为 0） |
| 3 | 充电站当前电价（非充电站为 0） |
| 4 | 充电站当前负荷率（非充电站为 0） |
| 5 | 配电网节点电压（标幺值） |
| 6 | 时间步编码（归一化） |
| 7 | 分时电价系数 |
| 8 | EV 指示位（当前决策 EV 所在节点为 1，其余为 0） |

**奖励函数：** 综合考虑充电功率收益（正向）、电压偏移惩罚、队列长度惩罚和线路损耗惩罚。

## `evaluation/` 评估目录

### `evaluation/metrics.py`
**评价指标计算模块**，提供用户侧和电网侧的量化评估指标。

| 类 | 说明 |
|---|---|
| `CostParams` | 集中管理成本参数：时间价值（VOTT = 30 CNY/h）、步时长、购电基准价格 |
| `UserMetrics` | 从 EV 对象列表提取用户侧指标，包括行驶/等待/充电时间成本、充电费用、总成本、平均等待时间等 |
| `GridMetrics` | 从电网历史数据提取电网侧指标，包括配电网购电成本和累计电压偏移量 |
| `Evaluator` | 聚合类，在仿真过程中持续记录电网状态，episode 结束后一次性输出完整评估报告 |

### `evaluation/run_evaluation.py`
**评估入口脚本**，训练完成后运行，支持两种模式：

- **随机策略基线**（`use_random=True`）：以随机动作为基准，评估未经训练时的性能
- **DQN 策略评估**（`use_random=False`）：加载 `model/trained_dqn.pth`，以 ε=0 纯利用模式跑评估 episode

运行方式：
```bash
python evaluation/run_evaluation.py
```
结果输出至 `evaluation/results/evaluation_report.json`。

### `evaluation/results/evaluation_report.json`
评估结果的 JSON 格式输出文件，包含多轮评估的用户侧与电网侧指标均值。

## `visualization/` 可视化目录

### `visualization/run_training_with_viz.py`
**带可视化功能的训练入口**，与 `train.py` 中的训练逻辑相同，
额外集成了 `TrainingVisualizer` 进行训练过程数据记录与图表生成，
是推荐的训练运行方式。

运行方式：
```bash
python visualization/run_training_with_viz.py
```

### `visualization/visualize_training.py`
**训练过程可视化工具类 `TrainingVisualizer`**，负责：
- 逐 episode 记录奖励、ε 值、平均队列长度、电网过载次数等数据
- 训练结束后绘制奖励曲线（含滑动窗口均值）、ε 衰减曲线
- 将训练数据序列化保存至 `visualization/results/training_data.json`
- 生成训练摘要文本 `visualization/results/training_summary.txt`

### `visualization/visualize_network.py`
**交通网络状态可视化类 `NetworkVisualizer`**，以 NetworkX + Matplotlib 渲染当前仿真步的图状态：
- 节点颜色表示 EV 数量，充电站节点使用特殊形状标注
- 显示各节点的队列长度、电价信息
- 可将图像保存为文件或直接展示

### `visualization/visualize_performance.py`
**训练后性能分析类 `PerformanceAnalyzer`**，读取 `training_data.json`，
计算并绘制：
- 奖励的最大值、最小值、均值、标准差和趋势斜率
- 奖励与 ε 的对比折线图
- 各辅助指标（队列长度、过载次数等）的历史曲线

### `visualization/visualize_simulation.py`
**仿真动画演示模块**，以启发式策略（队列 + 电价加权最小的站）驱动 EV 决策，
使用 Matplotlib Animation 生成逐步动态可视化动画，
可输出 GIF/MP4 文件，用于直观展示仿真场景。

### `visualization/results/`
可视化和训练过程输出目录：
- `training_data.json`：JSON 格式的每 episode 训练数据（奖励、ε、队列等）
- `training_summary.txt`：训练结束后自动生成的文字摘要报告
