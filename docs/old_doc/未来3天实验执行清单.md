# 未来 3 天实验执行清单

## 总目标

当前 `FedDQN` 已经从早期失败状态进入“接近 Greedy、具备论文主线潜力”的阶段。

接下来最重要的不是继续大改模型，而是：

- 锁定当前主配置
- 跑完整规模主结果
- 做多 seed 稳定性验证
- 补基线与消融实验

## 当前主配置

本轮默认使用：

- 事件触发决策
- 受限重规划（每次充电任务最多 1 次）
- 18 维堵站风险状态特征
- 固定评估 seed
- reward 归一化与截断
- 当前训练节奏与 batch size 配置

后续实验阶段，不建议再改核心逻辑。

---

## 第 1 天：正式完整训练与主结果确认

### 任务 1：完整规模训练

命令：

```bash
python main.py train-real
```

目标：

- 获取当前主配置下的完整规模模型
- 保留训练日志与 checkpoint

### 任务 2：完整规模评估

命令：

```bash
python main.py evaluate
```

目标：

- 获取正式规模四方对比结果
- 更新 `docs/评估结果记录.md`

### 第 1 天重点观察指标

- `avg_wait_time_min`
- `abandoned_evs`
- `total_charging_cost_per_veh`
- `evs_in_line`
- `end_evs_in_line`

如果完整规模结果仍能接近或略优于 `Greedy`，说明当前主线可继续推进。

---

## 第 2 天：多 Seed 稳定性验证

### 任务 1：跑 3 个训练 seed

目标：

- 验证当前配置不是偶然跑好一次
- 观察均值与波动范围

建议至少做：

- 3 个训练 seed

更理想：

- 5 个训练 seed

### 任务 2：汇总均值

重点统计：

- `avg_wait_time_min`
- `abandoned_evs`
- `total_charging_cost_per_veh`
- `incomplete_evs`

### 判断标准

如果 `FedDQN` 在多 seed 均值上：

- 接近或略优于 `Greedy`
- 且波动不大

则当前方案已具备论文主结果基础。

---

## 第 3 天：补基线与做消融

### 任务 1：补论文相关基线

优先建议：

- `FedAvg without Prox`
- `Nearest + Shortest Queue` 风格启发式

目标：

- 证明不是“随便一个启发式都能做到”
- 证明 FedProx / 当前策略设计确实有贡献

### 任务 2：做消融实验

优先建议的 3 个消融：

1. 去掉受限重规划
2. 去掉新增堵站风险状态特征
3. 去掉 reward 归一化

目标：

- 证明当前版本各个改动分别带来了什么贡献

---

## 当前不建议做的事情

以下动作暂时不建议优先：

- 再次大改网络结构
- 继续频繁调整 reward 系数
- 反复追求单次结果比 `Greedy` 高 1% 到 2%

原因：

- 现在最缺的是实验体系完整性，不是继续堆工程改动
- 单次结果接近 `Greedy` 已足够说明模型开始有效
- 论文更需要“稳定 + 可解释 + 有消融”

---

## 当前阶段的核心目标

从现在开始，目标应从：

- “继续把 FedDQN 调得更强”

转为：

- “证明当前 FedDQN 稳定、有效，并解释为什么有效”

---

## 一句话执行原则

锁配置、跑 full、做多 seed、补基线、做消融。
