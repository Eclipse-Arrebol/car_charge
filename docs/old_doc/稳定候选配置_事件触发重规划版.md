# 稳定候选配置：事件触发 + 受限重规划版

## 适用目的

本配置用于当前阶段的真实路网联邦训练主线，目标是：

- 保留事件触发决策，避免每步重规划导致训练过慢
- 在排队明显恶化时允许有限重规划，减少热门站扎堆
- 强化排队相关奖励，避免策略只学会“就近充电”
- 提高训练吞吐，让 4090D + 服务器配置能稳定跑完整实验

## 当前建议命令

完整版训练：

```bash
python main.py train-real
```

快速验证：

```bash
python main.py train-real --medium
python main.py evaluate --medium
```

## 当前稳定候选配置

### 决策机制

- 低电量事件触发首次规划
- 非每步重规划
- 每次充电任务最多允许 1 次途中重规划

途中重规划触发条件：

- 仅在 `MOVING_TO_CHARGE` 过程中检查
- 当前目标站排队接近爆满，或预测等待时间过高
- 同时存在明显更优替代站

### 训练节奏

完整版默认：

- `episodes = 500`
- `steps_per_episode = 500`
- `batch_size = 256`
- `step_local_train_steps = 1`
- `step_train_interval = 2`

`medium` 默认：

- `episodes = 100`
- `steps_per_episode = 200`
- `batch_size = 256`
- `step_local_train_steps = 1`
- `step_train_interval = 4`

### reward 稳定化

训练时写入 replay 的 `mixed_r` 处理：

- `mixed_r /= 50.0`
- clamp 到 `[-20.0, 5.0]`

作用：

- 缩小 reward 量级
- 缓解 Q 学习不稳定
- 减少极端拥堵样本对训练的破坏

### 排队导向 shaping

当前训练 reward 更强调排队与服务时间：

- 强化 `queue_time_h` 惩罚
- 强化 `service_time_h` 惩罚
- 降低对 `trip_time_h` 的偏好
- 加入 `grid_load_ratio` 轻惩罚

环境全局 reward 中也提高了 `queue_cost` 权重。

## 当前阶段的判断标准

不要只看总 reward。

更建议优先看：

- `avg_wait_time_min`
- `abandoned_evs`
- `evs_in_line`
- `end_evs_in_line`
- `total_charging_cost_per_veh`

训练日志中可重点观察：

- `avg_mixed_r`
- `avg_queue_h`
- `avg_real_queue_h`
- `avg_service_h`
- `avg_real_service_h`

## 当前实验现象总结

相较于旧版：

- FedDQN 不再严重排队崩溃
- `avg_wait_time_min` 明显下降
- `abandoned_evs` 明显下降
- `evs_in_line` 已接近 Greedy
- `total_charging_cost_per_veh` 已具备优势

仍待继续优化的点：

- 等待时间整体仍略逊于 Greedy
- 放弃充电数量仍高于 Greedy
- 出行时间优势有所减弱，说明当前策略更偏避堵

## 建议的后续策略

当前不建议再做大改结构。

建议顺序：

1. 先基于这套配置继续完整训练与多 seed 评估
2. 再做小幅 reward 系数微调
3. 最后再决定是否需要进一步改状态或重规划规则
