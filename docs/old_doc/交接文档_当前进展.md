### 2.4 训练稳定性修复

已完成：

- `evaluation/run_evaluation.py` 已修复评估时漏传 `action_mask`

### 2.5 联邦训练排查与修复

已确认过：

- replay buffer 没有在联邦轮次间被清空
- epsilon 不会在聚合后被错误重置为 `1.0`
- FedAvg 参数分发链路基本是通的

已做的最小修复：

## 3. 已验证结果

### 3.1 debug 评估结果（修复后）

曾验证通过的结果：

- `Random avg_wait_time_min ≈ 30.38`
- `DQN avg_wait_time_min ≈ 11.90`
- `FedDQN avg_wait_time_min ≈ 27.75`

结论：

- `FedDQN` 已经不再比 `Random` 更差
- 但仍明显落后于 `DQN`

### 3.2 训练日志观察

在 `train-real --debug` 中可看到：

- 初期部分 client `optimizer_state_entries=0`
- 之后两侧 client 都能到 `optimizer_state_entries=54`

说明：

- 联邦 client 后期已经不是完全空跑
- 参数更新确实发生了
